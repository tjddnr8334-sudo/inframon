# 공통 — 데이터 계약과 파이프라인 (어느 엔진을 건드리든 먼저 읽기)

> 엔진 내부 구현은 자유지만, `project.h5` 입출력 계약을 어기면 다운스트림이 **조용히** 망가집니다. 엔진 작업 전 이 규칙을 확인하세요.
> 전체 문서: [`../개발_맥락_맵.md`](../개발_맥락_맵.md) §2·§3

## 핵심 철학
엔진은 서로를 직접 호출하지 않는다. 오직 **contracts(Pydantic 모델 + 단일 `project.h5`)** 를 통해서만 데이터를 주고받는다. 덕분에 STUB를 mock 삼아 엔진을 독립 개발하고 핫스왑할 수 있다.

## 계약 backbone — `src/inframon/contracts/`
- **`schema.py`** — 7개 Pydantic 모델(`CVGeometry`, `CVOutput`, `InSAROutput`, `PINNOutput`, `FRAMWarning`, `FRAMOutput`) + 표준 상수 튜플 `MEMBER_TYPES`, `FRAM_FUNCTIONS`.
- **`io.py`** — `ProjectStore`가 `project.h5`를 감쌈: `write_array/read_array/write_meta/read_meta/has_meta/require_group`.

## 계약이 작동하는 4가지 메커니즘
1. **`*_ds` 규약** — 큰 배열은 Pydantic에 직접 안 담고 HDF5 경로 문자열(`roi_mask_ds='/cv/roi_mask'`)만 보유. 모델은 가볍고 JSON 직렬화 가능.
2. **디스크-경계 타입 검증** — `write_meta`=`model_dump_json`, `read_meta`=`model_validate_json` 왕복. 잘못된 스키마는 로드 시 `ValidationError`.
3. **표준 라벨 단일 진실 소스** — `MEMBER_TYPES=('deck','pier','abutment','bearing')`, `FRAM_FUNCTIONS=('thermal','load','bearing','foundation')`.
4. **흐름 의존성이 시그니처에 박힘** — `run_insar(store, cv)` → `run_pinn(store, insar)` → `run_fram(store, insar, pinn)`.

## project.h5 구조
```
project.h5
├── /cv     (attrs['meta']=CVOutput JSON)   + roi_mask, member_*, grid_density
├── /insar  (attrs['meta']=InSAROutput)     + los, longitudinal, xyz, ...
├── /pinn   (attrs['meta']=PINNOutput)      + comp_*, strain, V_*, ...
└── /fram   (attrs['meta']=FRAMOutput)      + R_ij, amplification, CRI, ...
```

## 파이프라인 흐름 (orchestrator/pipeline.py, mode='w')
```
cfg → run_cv → CVOutput → run_insar → InSAROutput → run_pinn → PINNOutput → run_fram → FRAMOutput
```
| 단계 | 입력 | 출력(핵심) | 다음이 쓰는 것 |
|---|---|---|---|
| CV | cfg | roi_mask, member_label, grid_density | grid_density, member_label, azimuth |
| InSAR | CVOutput | los[N,M], longitudinal[N,M], dates | longitudinal, dates, member (+xyz→FRAM) |
| PINN | InSAROutput | comp_*, V_func_series[4,M] | V_func_series, comp_*→FRAM |
| FRAM | InSAR+PINN | CRI[N,M], warning | — (대시보드 소비) |

## 계약을 깨지 않는 규칙 (전 엔진 공통)
- ⚠️ **`V_func_series_ds`는 `[4,M]`이고 행 순서가 `('thermal','load','bearing','foundation')`** — FRAM이 이 순서로 R_ij/A를 계산한다. 어기면 조용히 오작동.
- HDF5 attribute **64KB 한계** — meta JSON에 큰 좌표를 직접 넣지 말 것(`CVGeometry.centerline` 예외 주의).
- `*_ds` 경로와 실제 데이터셋 경로 정합성은 **강제 안 됨** → 오타 시 `read_array`에서 `KeyError`.
- `read_meta`는 model 타입을 교차검증 안 함 → 올바른 모델 클래스를 직접 전달할 것.
- `contracts/__init__.py`는 0바이트 → `from inframon.contracts.schema import ...` / `.io import ...` 풀패스 임포트만 가능.
- orchestrator는 `mode='w'` 전체 덮어쓰기 — **증분/부분 실행 없음**(매번 4단계 전부).
