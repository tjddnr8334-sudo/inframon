# inframon 시스템 개요 — 체계 구조 · BMAPS 동기화 · 독자성

> 논문의 "시스템 구성" 절/부록 초안으로도 활용할 수 있도록, 코드(`src/inframon/api/`)와
> 설계 문서를 종합해 정리한 상위 개요. 세부 설계는 `docs/` 하위 문서를 참조.

---

## 1. 전체 체계 구조

핵심은 **"계약(`project.h5`)을 성역으로 둔 허브 + 4엔진 핫스왑"** 구조다. 모든 계산 모듈은
결과를 단일 HDF5 계약에 기록하고, 상위 계층(서빙·대시보드·핸드오프)은 그 계약을 **읽어 변환만**
한다. 계약(`src/inframon/contracts/`)은 골든 회귀 테스트로 보호되어 함부로 바뀌지 않는다.

```
                    ┌─────────────── 계약(성역) ───────────────┐
 위성/외부 데이터 →  │  contracts/{schema.py, io.py}  ⇄  project.h5 │  ← 단일 허브(Pydantic+HDF5)
                    └───────────────────┬───────────────────────┘
   ┌────────────── 4엔진 (stub ⇄ real 핫스왑, 골든회귀 보호) ──────────────┐
   │  CV → InSAR → PINN → FRAM   ( --engine X=real )                     │
   └───────────────────┬────────────────────────────────────────────────┘
        orchestrator/ (pipeline·engines·incremental 재개)
                        │  project.h5 갱신
        ┌───────────────┼────────────────────────────┐
     대시보드          사이드카 API              KAIA 핸드오프
   (Streamlit)     (--serve-api, FastAPI)   (CSV / VLM 패키지)
                        │
                     BMAPS 탭
```

### 엔드투엔드 파이프라인 (선별 A–E → 처리 F → 해석 G → 핸드오프 H → 연동 I)

| 단계 | 내용 | 담당 모듈 / CLI |
|---|---|---|
| **A** 교량 선별 | OSM 교량 확정(bbox·길이·폴리라인) | `osm_bridge.confirm_bridge` |
| **B** SLC/트랙 선별 | ASF 장면 검색 · 궤도 선정 | `slc_search.{search_slc, select_track}` |
| **C** master 선정 | ERA5(Open-Meteo) 건조·저습일 선정 | `era5_master` |
| **D** 조건 게이팅 | 교량 InSAR 신뢰성 조건 검증 | `--insar-conditions` |
| **E** 처리 번들 | 형식별 SARvey config 생성 | `--make-sarvey-config` |
| **F** 실 SLC 처리 | ISCE2 → MiaplPy → **SARvey** (WSL2/Linux) | `scripts/wsl_sarvey/*` → Track H5 |
| **G** 인제스트 + 4엔진 | CV → InSAR → PINN → FRAM → CRI | `--demo --insar-source track.h5 --engine …=real` |
| **H** KAIA 핸드오프 | 변위 CSV · VLM 입력 패키지 | `--export-csv`, `--export-vlm` |
| **I** BMAPS 연동 | REST 사이드카 엔드포인트 | `--serve-api --registry …` |

성숙도: **A–E 완료**(정자교 실 장면 검증), **F 진행 중**(WSL2 코레지/간섭), **G–I 코드 완성**.
실 사례 시연치는 정자교 Sentinel-1 **1,072점 × 201에폭**, 해석해 검증 **오차 < 0.1%**,
테스트 파일 43개 · 테스트 304개.

## 2. BMAPS와의 동기화

**한 줄 요약: 단방향(read-only) REST "사이드카" — BMAPS가 필요할 때 당겨가고(pull),
inframon으로의 역기록은 없다.** (근거: `src/inframon/api/app.py`, `docs/Bmaps_연동_인터페이스.md`)

```
┌───────────────── BMAPS (KICT 교량 SHM) ─────────────────┐
│ [성능평가][노후도]…[★ InSAR 위성 변위 분석] ← 새 탭(프론트) │
└───────────────────────────┬──────────────────────────────┘
                            │ REST/JSON, 교량ID로 조회 (pull)
              ┌──────────────▼───────────────┐
              │ inframon-api (FastAPI 사이드카) │  --serve-api
              └──────────────┬───────────────┘
                            │ 매 요청마다 최신 읽기(read-only)
              ┌──────────────▼───────────────┐
              │ 교량별 project.h5 (계약)        │
              └──────────────────────────────┘
```

### 동기화 성격

- **방향**: 단방향. BMAPS → 사이드카 GET 요청, 사이드카 → `project.h5` 읽기. **write-back 없음.**
- **실시간성**: 파이프라인이 `project.h5`를 갱신하면 다음 요청에서 즉시 반영(배치 ETL 아님).
  사이드카는 매 요청마다 `project.h5`를 새로 연다(`app.py`의 `_store`).
- **경계**: `contracts/`는 성역 — API는 **읽어 변환만** 하고 새 계산은 하지 않는다.
- **매핑 키**: `bridge_registry.json`이 **BMAPS 교량관리번호 ↔ project.h5**를 1:1 연결
  (모든 엔드포인트의 경로 파라미터 `bridge_id`).

### 규약 변환 (`src/inframon/api/transform.py` — 오해 방지 지점)

- 좌표: 내부 `EPSG:5179` → API **WGS84(lat,lon)** (BMAPS가 5179 타일이면 `to_crs=SRC_CRS`로 변환 생략)
- 단위: 변위는 계약상 **이미 mm** — 그대로 통과(과거 ×1000 과대보고 버그 수정됨)
- 날짜: epoch days → **ISO YYYY-MM-DD**
- 부재: 정수 라벨 → `deck | pier | abutment | bearing`

### 엔드포인트 (`GET /api/v1/…`, 전부 읽기 전용)

| UI 요소 | 엔드포인트 |
|---|---|
| 헬스체크 | `/health` |
| 교량 핀/목록 | `/bridges` |
| 상단 경보·요약 | `/bridges/{id}/insar/summary` |
| 지도 변위 측점 | `/bridges/{id}/insar/points(.geojson)` `?metric=los\|longitudinal&date=latest\|<idx>` |
| 측점 클릭 시계열 | `/bridges/{id}/insar/points/{pid}/series` |
| CRI 추세 | `/bridges/{id}/insar/cri` |
| 함수망 진단(N-K) | `/bridges/{id}/insar/function-network` |
| KAIA CSV | `/bridges/{id}/insar/export.csv` |
| VLM 패키지 | `/bridges/{id}/insar/vlm-package.zip` `?figures=` |

오류 규약: `404`(교량/산출물 없음) · `409`(schema_version 불일치) · `503`(읽기 실패),
본문은 `{"error":{"code","message"}}`로 통일. CORS는 BMAPS 도메인만 허용(`--cors-origin`).
프론트 통합 데모: `examples/bmaps_tab/`(Leaflet · VWorld/OpenLayers · React, framework-agnostic ES 모듈).

## 3. 프로그램의 독자성

**기반 InSAR 도구(ISCE2 / MiaplPy / SARvey)가 멈추는 지점 = LOS 변위 시계열까지.**
inframon의 독자성은 그 위에 **구조 물리(PINN) · 시스템 안전(FRAM) ·
플랫폼(계약 · 대시보드 · API · 핸드오프)** 을 얹은 데 있다.

| 계층 | inframon 고유 기여 | 기반 도구엔 없음 |
|---|---|---|
| **선별 A–E** | OSM 교량 확정 · ASF 트랙 선별 · ERA5 master · 신뢰성 조건 게이팅 · 형식별 SARvey config | SARvey는 수동 ROI bbox만 |
| **IO 어댑터** | ISCE2/MiaplPy/SARvey/MintPy/StaMPS → **통일 Track H5** 인제스트 | SARvey는 ISCE2 출력 한정 |
| **CV↔InSAR 정합** | 픽셀↔월드 아핀 · CRS 재투영 · DEM 고도 폴백 · 부재별 적응 점밀도 | SARvey는 균일 grid |
| **PINN 구조 역산** | Euler-Bernoulli **PDE autograd loss** · **형식별 PDE**(거더/사장/아치/현수) · **절대 EI 식별** · FEM 모달 검증 · 온도(α·L·ΔT)·교통 외생 | SARvey는 LOS만 출력 |
| **성분 분해** | 변위 → thermal / load / settle / anomaly 분리 | SARvey는 변위를 단일체로 |
| **FRAM 시스템 안전** | 6기능 함수망 · **N-K 공명(스펙트럴 반경)** · **isotonic 보정** · 4단계 경보 | 대응물 없음 |
| **계약·핫스왑** | 단일 `project.h5` 허브 · stub↔real 교체 · 증분 재개 · 골든 회귀 | 워크플로우에 없음 |
| **대시보드/API/핸드오프** | Streamlit 3탭 · 함수망 다이어그램 · REST 엔드포인트 · KAIA CSV/VLM 패키지 | SARvey는 QGIS·스택 출력만 |

한 줄 요약: **SARvey는 LOS 변위까지, inframon은 그 위에 구조물리(PINN)·시스템안전(FRAM)·
플랫폼(계약·대시보드·API·핸드오프)을 얹은 것.**

### 책임 경계 (정직성)

CRI·경보는 inframon 내부 물리 지표이며 **코드 기반 안전 판정이 아니다** — 최종 판정은
VLM + 시방 권한의 몫(VLM 패키지 manifest의 `risk_disclaimer`에 명시). 현장 실측(AVT·GNSS)·
상용 FEM 교차검증·실 붕괴 라벨 검증은 미수행(연구 프로토타입).
