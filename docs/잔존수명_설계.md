# 잔존수명(RSL) 추정 설계 — PINN 기반

> **상태: P1(사용성 채널) 구현됨 · P2~P4 미구현.** `--remaining-life` 로 동작하고
> `/life` 에 기록된다. 구현 현황의 신뢰원은 [`개발_맥락_맵.md`](개발_맥락_맵.md) 다.
>
> **구현하며 바뀐 것(실 데이터에서 드러난 것):**
> - 부등침하에 **최소 이격(기본 5m)** 을 뒀다. 각변위 한계 1/500 은 인접 지지점 간 규정이라
>   1m 떨어진 쌍에 적용하면 각변위가 발산해(2mm/yr 차이 → 잔존수명 1년) 과대평가된다.
> - 각변위를 **자기 추세가 유의한 점에만** 귀속시킨다. 안 그러면 고립 이상치 한 점이 주변
>   정지점에까지 짧은 잔존수명을 전파해 가짜 군집을 만들고 공간 응집 규칙이 무력화된다.
> - 지간을 점군 범위로 *추정*만 한 경우 상판에 **L/800 을 적용하지 않는다**. 실 SARvey 점군은
>   주변 건물까지 포함해 신장이 1519m(교량 ~120m)로 나오고, 그러면 한계가 1.9m 가 되어
>   상판에서 아무것도 검출되지 않는다. 모르는 값으로 만든 큰 한계는 안전이 아니라 검출 불능이다.
> - 교량 대표값과 함께 **지배 군집의 좌표·규모·속도(핫스팟)** 를 기록한다. 숫자 하나로는
>   어디를 점검할지 알 수 없어 스크리닝으로서 쓸모가 없다.
> - **LOS→연직 투영**을 넣었다(§2.4). 사용성 한계는 연직 규정인데 LOS 를 그대로 쓰고 있었고,
>   그 결과 잔존수명이 1/cos θ ≈ **1.29배(39°) 낙관적**이었다. 정자교 재실행: 1.7년 → 1.2년.
> 이 문서는 "교량 잔존수명을 PINN 으로 낸다"를 **어떤 데이터로·어떤 한계상태로·어느 신뢰도로**
> 할 것인지 확정하기 위한 설계다.

## 0. 결론 요약

1. **잔존수명은 하나의 수가 아니다.** 물리적으로 다른 한계상태 3~4개가 각각 다른
   시간을 준다. 이를 하나로 뭉개면 숫자는 예뻐지지만 근거가 사라진다.
   → **채널별로 따로 계산하고, 지배(최소) 채널을 이름과 함께 표시**한다.
2. **위성 InSAR + PINN 이 실제로 측정하는 것은 운동학(변위·속도·곡률→EI)뿐이다.**
   부식률·프리스트레스 손실·균열폭·중성화는 **측정하지 못한다**. 따라서 채널을
   **측정기반(measurement-driven)** 과 **가정기반(model-driven)** 으로 나누어 표기한다.
3. **대부분의 건강한 점은 잔존수명이 정의되지 않는다**(침하속도가 0과 유의하게 다르지 않음).
   이를 "잔존수명 500,000년"으로 표시하면 안 되고 **검열(censored, `> horizon`)** 로 다뤄야 한다.
   이 처리가 이 기능의 성패를 가른다.
4. 4엔진(`ENGINE_NAMES`)은 건드리지 않는다. 잔존수명은 FRAM 이후의 **후처리 스테이지**로
   붙이고(`--remaining-life`, opt-in), PINN 은 시간분해 EI 하나만 추가로 뱉는다.
   → 골든 회귀·기존 계약 불변.

## 1. 한계상태(채널) 정의

| # | 채널 | 한계상태 | 근거 데이터 | 성격 | 우선순위 |
|---|---|---|---|---|---|
| B | **사용성**(serviceability) | 침하·처짐이 허용치 초과 | `/insar/velocity_mm_yr`, `comp_settle`, `vsens_deflection`, `deck_deflection` | **측정기반** | **P1** |
| A | **강성열화**(stiffness) | `EI(t)/EI₀ < r_limit` (기본 0.80) | PINN `EI_series`(신규) | 측정기반, **장기 관측 필요** | P2 |
| C | **피로**(fatigue) | Miner 누적손상 D ≥ 1 | 교통량 × 영향선 응력범위 × S-N | **가정기반** | P3 |
| D | **내구성**(carbonation/chloride) | 중성화·염화물이 철근 위치 도달 | 피복두께·노출등급·w/c (점검자료) | **가정기반**, 위성 무관 | P4(선택) |

**대표 잔존수명 = min(활성 채널)**, 지배 채널명·각 채널 신뢰구간을 항상 동반 표기.

### 1.1 왜 채널 C(피로)를 InSAR 시계열로 계산하면 안 되는가

Sentinel-1 은 6~12일 간격 샘플이다. 피로는 **차량 1대당 응력 사이클**(Hz 대역)이 누적되어
발생한다. InSAR 시계열에 rainflow 계수를 돌리면 존재하지 않는 사이클을 세게 되고 결과는
물리적으로 무의미하다. 따라서 채널 C 는 반드시
`일교통량(EX API) × 대형차 혼입률 × 영향선 기반 1대당 응력범위(PINN 단면·EI) × S-N(상세등급)`
경로로 계산하고, **UI 에 "설계코드 추정치(측정 아님)"로 명시**한다.

### 1.2 외삽 전에 반드시 제거해야 하는 것

열팽창·계절 성분이 남은 신호를 외삽하면 관측창의 계절 위상에 따라 속도가 통째로 바뀐다.
채널 B 는 **반드시** PINN 성분분해 후의 `comp_settle`(+`comp_anomaly`) 또는
`--insar-thermal` 로 열성분을 분리한 `/insar/velocity_mm_yr` 를 입력으로 쓴다.
원 `los_ds` 직접 외삽 금지.

## 2. 채널 B — 사용성 (P1, 이번에 실제로 만들 것)

### 2.1 점별 계산

각 점 i 에 대해 침하 시계열 `d_i(t)` [mm] (열성분 제거됨):

1. **강건 회귀** — Theil–Sen 기울기 `v_i` [mm/yr] + 부트스트랩/해석적 표준오차 `σ_i`.
   (OLS 는 InSAR 이상치(unwrapping error)에 취약. 기존 `atmo.temporal_decompose` 의
   최소자승과 달리 여기선 강건 추정이 필요.)
2. **유의성 검정** — `|v_i| < z_{0.95}·σ_i` 이면 **열화 신호 없음** → `RSL_i = +inf` (검열).
3. **여유(margin)** — `m_i = limit_i − |d_i(t_now)|`.
   `m_i ≤ 0` 이면 이미 초과 → `RSL_i = 0` + `이미 한계 초과` 플래그.
4. **잔존수명과 보수적 하한**
   ```
   RSL_i        = m_i / v_i
   RSL_lower_i  = m_i / (v_i + z_{0.95}·σ_i)     ← 보고용 기본값
   ```
   **표시 기본은 하한**이다. 점추정만 보여주면 과신을 유발한다.
5. **가속 검정** — 2차항 `d = a + bt + ct²` 적합, `c` 가 유의하고 부호가 악화 방향이면
   2차 해를 쓰고 `가속열화` 플래그. (관측창이 짧으면 검정력 부족 → 플래그만 남기고
   선형 하한과 min 결합.)

### 2.2 교량 대표값 — 최솟값을 쓰지 않는다

단일 최소점은 노이즈 하나에 지배된다. **공간 응집 규칙**을 쓴다:

> 반경 `r`(기본 = 점간 중앙거리 × 3) 안에서 **RSL 이 하위 임계 이하인 점이 k개(기본 3) 이상
> 모인 군집**만 유효 열화로 인정하고, 그 군집들의 대표 RSL 중 최솟값을 교량 RSL 로 한다.
> 유효 군집이 없으면 교량 RSL = `> horizon`(검열).

이는 FRAM 의 `spatial_prop` 과 같은 사상(고립점은 결함이 아니다)이며, 이미
`insar` 계약의 `xyz_ds` 로 거리 계산이 가능하다.

### 2.4 LOS 는 연직이 아니다 — 반드시 되돌려야 한다

침하 허용치(25mm)도 부등침하 각변위(1/500)도 **연직량** 규정이다. 그런데 단일 궤도
InSAR 가 재는 것은 위성 시선 성분뿐이다.

```
LOS ≈ d_vertical·cos θ − d_horizontal·sin θ·(방위 성분)
```

Sentinel-1 IW 입사각 θ 는 29°~46°(대표 39°)라 `cos θ ≈ 0.69~0.87`. LOS 를 연직인 척
쓰면 변위가 그만큼 **과소평가**되고 잔존수명은 역수만큼 길어진다 — 39° 면 **29% 낙관적**.

그래서 `life/geometry.py` 가 **단일 궤도 연직 가정**으로 되돌린다.

```
d_vertical = d_LOS / cos θ
```

- 입사각은 상류 Track H5 에 있고(`incidenceAngle`) 이제 `/insar/incidence_deg` 로
  **보존된다**(계약 1.4). 이전에는 종축 분해에만 쓰고 버려서 하류가 알 수 없었다.
- 없으면 39°(IW 대표값)를 가정하고 그 사실을 결과에 남긴다. 참값 33°에 39°를 쓰면
  오차 8% 정도로, 투영을 안 할 때(29%)보다 훨씬 낫다.
- 라디안이 들어오는 등 범위(10°~75°) 밖 값은 신뢰하지 않고 대표값으로 대체한다.
- **`/insar/vertical` 에는 이 투영값을 넣지 않는다.** 그 계약 필드는 asc+desc **실제 기하
  분해**를 뜻하고 소비자는 그렇게 신뢰한다. 투영은 소비 측에서 가정을 밝히고 한다.
- 가정이 틀리는 경우(수평 이동이 지배적인 사면·측방유동)에는 연직량이 과대평가된다.
  진짜 해법은 asc+desc 융합이고, 융합 연직이 있으면 이 투영은 아예 건너뛴다.

열성분 제거는 **투영 전에** 한다 — PINN `comp_thermal` 이 LOS 기하 값이기 때문이다.

### 2.3 한계값 `limit_i`

부재별로 다르다(`member_ds` 로 분기):

| 부재 | 한계 | 기본값 | 출처 |
|---|---|---|---|
| deck | 활하중 처짐 | `L/800` | 도로교설계기준(한계상태설계법) 사용성 |
| pier / abutment | 절대 침하 | 25 mm | 사용자 지정 필수(지반조건 의존) |
| 인접 지점 | **부등침하** | `1/500 × 지간` | 기울기 기반 — 절대침하보다 구조적으로 지배적 |
| bearing | 이동량 | 가동단 설계이동량 | 사용자 지정 |

기본값은 **반드시 출처 라벨과 함께** `assumptions` 에 기록하고 UI 에 노출한다.
부등침하는 절대침하보다 교량에 치명적이므로 **인접점 기울기 채널을 별도로 계산**한다.

## 3. 채널 A — 강성열화 (P2)

### 3.1 PINN 변경점: EI 를 시간분해로

현재 `PINNOutput.EI_ds` 는 `[N]` — 관측창 전체에 대해 점당 한 값이라 **추세를 못 낸다.**
그런데 `pinn/real_engine.py` 의 절대 EI 식별은 이미 시점별로 계산한 뒤 평균한다:

```python
d4_vals = []
for tc in t_sub:                     # ← 시점별 루프가 이미 있다
    ...
    d4_vals.append(float(gg.detach().abs().mean().item()))
d4_hat = float(np.mean(d4_vals))     # ← 여기서 시간정보가 버려진다
```

따라서 **재학습 없이** 시점별 EI 를 얻을 수 있다:

```python
EI_series[k] = _identify_EI_from_pde(d4_vals[k], L_m, q_eff, w_scale_used * 1e-3)
```

→ 신규 Optional 필드 `EI_series_ds` `[M_sub]` + `EI_series_t_ds` `[M_sub]`(시점 인덱스).
`EI_ds`(기존 `[N]`)는 **불변** → 골든 회귀 안전, 스키마 `1.2 → 1.3`(하위호환 minor).

### 3.2 추정

`EI₀` = 기준값. 우선순위: ① 관측 초기 구간 중앙값, ② `geometric_EI`(설계 단면에서 계산,
이미 `pinn.inputs.geometric_EI_Nm2` 에 있음). ②를 쓰면 "설계 대비 현재 강성비"라는
훨씬 유용한 지표가 되지만, 식별 EI 와 기하 EI 사이의 계통 편차가 있으므로
**둘 다 계산해 함께 보고**하고 외삽은 ①(자기기준) 로만 한다.

`log EI(t)` 선형 회귀 → 열화율 `λ` [1/yr] → `RSL_A = ln(r_limit)/λ`, 하한 동일 방식.

### 3.3 게이팅 — 이 채널은 쉽게 거짓말을 한다

EI 식별은 `d4_hat`(4차 도함수) 에 의존하고 이는 노이즈에 매우 민감하다. 그래서:

- **관측기간 < 3년** → 채널 A 비활성 + `관측기간 부족` 사유 표시(값 자체를 내지 않음).
- `λ` 의 95% 신뢰구간이 0 을 포함 → 검열.
- `EI(t)` 의 변동계수가 임계 초과 → `식별 불안정` 으로 비활성.

이 게이트 없이 EI 추세를 노출하면 사용자는 노이즈를 열화로 읽는다.

## 4. 채널 C — 피로 (P3, 가정기반)

```
Δσ_truck  ← PINN 영향선(EI·단면·경계조건) × 설계활하중(DB-24, public_data.design_load)
N_cycles  ← 일교통량(EX API) × 대형차 혼입률 × 차로분배계수 × 경과연수
D         ← Σ n_i / N_i   (S-N, 상세등급 EC3 71/90/… 또는 AASHTO A~E′)
RSL_C     = (1 − D_now) / (dD/dt)
```

강재교(또는 강합성)에서만 활성. 상세등급은 **사용자 입력 필수**(자동 추정 불가).
결과는 `채널 성격: 설계코드 추정` 라벨 고정.

### 5.3 사람이 넣어야 함 — **이번에 새로 받아야 하는 것**

| # | 항목 | 단위/형식 | 없으면 | 쓰는 채널 |
|---|---|---|---|---|
| 1 | **설계공용수명** | 년 (기본 100) | 기본값+출처라벨 | 전체(상한) |
| 2 | **처짐 한계** | `L/800` 등 비 또는 mm | 기본 L/800 | B |
| 3 | **허용 침하 / 부등침하** | mm / 비(1/500) | 기본 25 mm, 1/500 | B |
| 4 | **강성 열화 허용비** `r_limit` | 0.80 등 | 기본 0.80 | A |
| 5 | **피로 상세등급** | EC3 71/90/…, AASHTO A~E′ | 채널 C 비활성 | C |
| 6 | **보수·보강 이력** | `year,type` CSV | 추세 단절을 열화로 오독 | A·B |
| 7 | **점검등급 시계열** | `year,grade` CSV | 최신 등급만 사용 | 검증·사전분포 |
| 8 | (콘크리트) 피복두께·노출등급·w/c | mm / 등급 / 비 | 채널 D 비활성 | D |

**6번(보수 이력)이 특히 중요하다.** 2020년에 받침을 교체했는데 그걸 모르면 알고리즘은
2020년 전후의 단차를 "급격한 열화"로 읽는다. 반대로 보수 후 구간만 써야 추세가 맞다.

### 5.4 위성으로 얻을 수 없음 — 가정으로 남고, 그렇게 표시해야 함

프리스트레스 손실 · 부식률 · 균열폭·균열도 · 실제 활하중 응력범위 · 재료 실강도 열화.

class RemainingLifeOutput(BaseModel):

## 8. 검증 계획 (실 붕괴 라벨이 없으므로)

1. **합성 주입 회귀** — `fram/synthetic.py` 방식으로, 알려진 침하율/EI 열화율을 주입한
   가상 교량을 만들고 RSL 이 주입값의 이론 해를 오차 내로 복원하는지.
2. **검열 동작** — 열화 없는 합성 교량에서 `censored_fraction ≈ 1`, 교량 RSL = `> horizon`.
   (이게 깨지면 모든 건강 교량에 가짜 수명이 붙는다 — 최우선 테스트.)
3. **강건성** — 이상치 주입(unwrapping error 모사) 시 Theil–Sen 결과가 OLS 대비 안정.
4. **골든 회귀** — `/life` 미사용 시 기존 파이프라인 수치 완전 불변.
5. **코호트 검사** — 건강 교량 여러 개(`fit_reference_range_from_projects` 와 같은 입력)에서
   RSL 분포가 대부분 검열되는지.

## 9. UI (대시보드 신규 탭)

- **헤드라인**: `잔존수명 ≥ 27년 (하한, 사용성 지배)` — 하한과 지배 채널을 항상 같이.
  검열이면 `> 50년(관측 구간에서 유의한 열화 없음)`.
- **채널 표**: 채널 / 성격(측정·가정) / 활성 여부 및 비활성 사유 / RSL / 하한.
- **점별 RSL 지도**: 기존 위험 지도 재사용(색 = RSL 하한, 검열점은 회색).
- **생존 곡선**: RSL 하한 대비 점 비율 — 어느 구간이 먼저 한계에 닿는지.
- **가정 패널(접기 아님, 항상 표시)**: 5.3 의 입력값과 출처, 5.4 의 "측정 불가" 목록.
- **관측 기반 고지**: 데크 점 희소 문제(매끈한 상판은 산란체 부족) 때문에 데크 RSL 은
  가상센싱 외삽임을 명시. 교대·교각 RSL 이 관측 근거가 더 강하다.

## 10. 정직한 한계 (문서·UI 양쪽에 고정)

- Sentinel-1 mm급·6~12일 샘플 → **연 0.5 mm 이하의 침하는 3년 관측으로도 유의하지 않다.**
  이 영역에서는 잔존수명의 **하한만** 의미가 있다("최소 N년 이상").
- 선형 외삽은 열화가 가속되면 낙관적이다. 2차 검정으로 일부 완화하나 관측창이 짧으면
  검정력이 부족하다.
- 매끈한 교량 상판은 자연 산란체가 적어 데크 점이 희소하다(실측 확인됨). 잔존수명 지도는
  교대·교각 위주가 되며, 데크는 가상센싱 외삽이다.
- 채널 C·D 는 설계코드·점검자료 기반 **추정**이고 위성 관측이 기여하지 않는다.
- **이 값은 안전 판정이 아니라 점검 우선순위 스크리닝이다.** README Status 배너와 동일한
  수위를 유지한다.
