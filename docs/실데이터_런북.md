# inframon 실데이터 런북

> 합성·데모 검증은 끝났다. 이 문서는 **실제 Sentinel-1 SLC → 변위 시계열 → CRI 위험맵**까지
> 사용자 데이터로 관통하는 절차다. 무거운 SAR 처리(ISCE2/MiaplPy/SARvey)는 **WSL2/Linux**에서,
> 나머지(선별·인제스트·해석·시각화)는 **Windows**에서 한다.

## 0. 준비도 진단 (먼저 실행)

```bash
python -m inframon --doctor                 # 환경(의존성·기능) 진단
python -m inframon --doctor <데이터루트>     # + InSAR 인벤토리 점검
python -m inframon --doctor <track.h5>      # + Track H5 preflight
```

- **필수**(numpy/h5py/pydantic)가 ❌면 코어가 안 돈다 → `pip install -e .`.
- 기능 게이트: `pinn_real`(torch), `crs_reprojection`(pyproj), `cv_geotiff_geo`(rasterio),
  `slc_search`(asf_search), `cv_transformer`/`cv_member_sam`(transformers). ⚪는 폴백으로 동작하나
  해당 기능은 비활성.

## 1. 교량 타깃 지정 (A·B) — Windows

지도에서 위치를 찍어 OSM 으로 교량을 확인하고 bbox 를 레시피로 저장.

- **대시보드**: `streamlit run src/inframon/dashboard/app.py` → ① InSAR 탭 → "교량 타깃 지정".
- 산출: `data/insar_recipe/bridge_target.json`.

## 2. SLC 검색·트랙 선별 (C·D) — Windows (asf_search)

교량 bbox 로 Sentinel-1 SLC(VV)를 검색해 취득 최다 트랙을 고른다. 공간 baseline ≤150 m.

- 대시보드 ① 탭 "SLC 검색·트랙 선별" 또는 `slc_search` API.
- 산출: `selection_criteria.json`, `track_selection.json`.

### 2-1. SLC 자동 다운로드 (자격증명만) — Windows

레시피(`processing_manifest.json`)의 선별 장면을 **자격증명만 넣으면** 프로그램이 자동으로 받는다
(순수 Python, WSL2 불필요). 자격 우선순위: 토큰 > 사용자/비번 > `~/.netrc`.

```bash
python -m inframon --download-slc data/insar_recipe \
  --earthdata-token <TOKEN>            # 또는 --earthdata-user <ID> --earthdata-pass <PW>
# 옵션: --slc-out <DIR>(기본 <recipe>/SLC) · --slc-limit N(처음 N장, 0=전체)
```

- SLC·VV·중복제거 후 전체(또는 N장) 다운로드. 이미 받은 `.zip` 은 자동 스킵(증분 재개).
- ⚠️ SARvey 시계열은 25장+·1년+ 권장 → 전체 선별 장면 확보 필요(수백 GB, 디스크 여유 확인).
  자격증명이 없으면 발급 안내 후 종료(`urs.earthdata.nasa.gov` 무료).

## 3. ERA5 master 선정 (E) — Windows (키 불필요)

취득일 강수·습도(Open-Meteo ERA5)로 가장 건조·저습한 날을 SARvey master 로.

- 산출: `master_selection_era5.json`.

## 4. SARvey 번들 생성 (F 준비) — Windows

레시피 4종 → 처리 매니페스트 + SARvey config.

```bash
# (먼저) 교량 InSAR 신뢰성 조건 게이팅 — SARvey 교량맞춤의 전제조건 점검(ready=0/not=1)
python -m inframon --insar-conditions data/insar_recipe
#   기하(종축 LOS 민감도·asc+desc 연직분리·입사각) · 시간샘플링(기간≥1년·장면≥25·계절·재방문)
#   · 산란체(길이·편파·수면마스킹) · 처리(격자·기준점·DEM오차) 14개 조건을 pass/warn/fail/unknown

python -m inframon --make-sarvey-config data/insar_recipe
# → processing_manifest.json (ISCE2/MiaplPy 스택) + sarvey_config.json (시계열)
```

> `--insar-conditions` 는 `--check-track`(처리 *출력* 검증)의 **입력측 짝** — 처리 *전에*
> 궤도-축선 기하·시간 샘플링이 교량 InSAR 에 충분한지 본다. 예: 단일궤도면 연직(처짐·침하)
> 미분리(G2 ⚠️), 종축이 LOS 에 수직이면 열팽창 관측 불가(G1 ⚠️) → 반대 궤도 추가 권고.

## 5. SAR 처리 (F 코어) — **WSL2/Linux 전용**

ISCE2/MiaplPy/SARvey 는 Windows 네이티브 불가. `scripts/wsl_sarvey/` 스크립트로 실행.

```bash
# (먼저) F코어 도구 설치 상태 감지 + 구축 안내(프로그램이 관리):
python -m inframon --insar-tools
#   ISCE2/MiaplPy/SARvey 존재를 WSL 에서 감지. 없으면 재현 레시피 2종 안내:
#     · conda     : bash scripts/wsl_sarvey/00_setup_env.sh
#     · container : docker build -t inframon-insar -f scripts/wsl_sarvey/Dockerfile .
#   ⚠️ 도구 바이너리는 리포에 담을 수 없다(각 수 GB·ISCE2 컴파일). 위 레시피가 리포에
#      버전 저장돼 있어 1회 구축하면 재사용. 컨테이너는 자체완결·이식가능.

# real 처리 계획(명령 시퀀스) 확인:
python -m inframon --insar-process real --recipe data/insar_recipe --work data/insar_work
```

WSL2(또는 컨테이너) 에서 순서대로 (conda env 별):
1. `00_setup_env.sh`(또는 `Dockerfile`) — isce2/mintpy/miaplpy/sarvey 환경 구축.
2. `10_download.sh` — SLC 다운로드(granule 이름 기반, VV·중복제거).
3. `20_stack_isce.sh` — ISCE2 topsStack(IW, multilook).
4. `30_miaplpy.sh` — MiaplPy phase-linking.
5. `40_sarvey.sh` (+`45_atmo.sh` 대기보정) — SARvey MTI 시계열.

> ⚠️ 외부 도구 실행 자체는 inframon 테스트 범위 밖(WSL2 실환경 필요). 어댑터·계약·셸 문법은
> 검증됨(`tests/test_wsl_adapters.py`). 도구 CLI 는 버전 의존이라 실행 시 확인 필요.

## 6. Export → Track H5 (F→G 경계) — WSL2 또는 Windows

처리 산출(시계열)을 inframon Track H5 로 변환. 도구별 어댑터:

| 처리기 | 어댑터 | 입력 |
|---|---|---|
| SARvey | `50_export_to_inframon.py` | 단일 H5(displacement/lat/lon/date/coh) |
| MiaplPy | `52_miaplpy_to_inframon.py` | timeseries+geometry+coherence 3파일 |
| MintPy | `54_mintpy_to_inframon.py` | timeseries(+affine attrs)+coherence |
| StaMPS | `56_stamps_to_inframon.py` | `.mat`(lonlat/ph_mm/day/coh_ps) |

```bash
python3 scripts/wsl_sarvey/52_miaplpy_to_inframon.py \
  --timeseries .../timeseries.h5 --geometry .../geometryRadar.h5 \
  --coherence .../temporalCoherence.h5 --out track.h5 --coh-thresh 0.6 \
  --bbox <minlon> <minlat> <maxlon> <maxlat>
```

산출 Track H5 표준 형식: `pixel_lonlat[N,2]` · `epochs[M]` · `los_mm[N,M]` · `coh[N]`
(+선택 `height[N]`, attr `crs`).

## 7. 투입 전 사전검증 (preflight) — Windows

```bash
python -m inframon --check-track track.h5    # ready=exit 0 / not=exit 1
```

- **차단(❌)**: 필수 데이터셋 누락·형상 불일치(N/M)·날짜 파싱 실패·점<2.
- **경고(⚠️)**: CRS 없음·고도 없음(z=0, 단 `--insar-dem` 으로 DEM 샘플 가능)·coherence 범위 이탈·일부 NaN·좌표가 경위도(→pyproj 필요).
- 차단이 있으면 어댑터 옵션(`--coh-thresh`/`--bbox`/`--*-key`)을 조정해 재생성.

## 8.5. GNSS 지상 근거 — 기준점 선정과 기준계

InSAR 는 **상대** 변위만 준다. 기준점을 어디 두느냐가 전 결과의 원점을 정하는데, 그 근거가
형식별 휴리스틱뿐이면 결과를 절대량으로 읽을 수 없다. NGL 상시 GNSS 가 그 선택을 관측으로
뒷받침한다([관측소 지도](https://geodesy.unr.edu/NGLStationPages/gpsnetmap/GPSNetMap_MAG.html),
데이터는 `DataHoldings.txt` + `.tenv3`).

```bash
# ① SLC 처리 전 — 이 교량에 지상 근거가 있는가
python -m inframon --gnss-anchor 37.3634,127.1090 --gnss-km 60

# ② 레시피 번들에 근거를 실어 SLC 처리로 가져간다
python -m inframon --make-sarvey-config <recipe_dir> --gnss-anchor-km 60
#    → processing_manifest.json 의 gnss_reference 블록

# ③ 결과가 나온 뒤 — 수직 대조 검증
python -m inframon --gnss-validate project.h5 --gnss-incidence 41.5 --gnss-km 60
```

앵커 후보는 **오래 관측됐고·연직으로 안 움직이고·산포가 작고·가까운** 순으로 점수를 매긴다.
관측 5년 미만이나 산포가 큰(장비 스텝) 관측소는 사유와 함께 제외한다.

**절대 타이는 GNSS 가 InSAR 발자국 안(≤2km)일 때만 정당하다.** 그보다 멀면 그 사이 지반이
같이 움직인다는 보장이 없으므로 자동 타이를 거부하고 **지역 연직 기준계 참고**로만 쓴다.
정자교는 최근접 SUWN 이 10.9km 라 여기 해당한다 — 기준점 선정 근거로는 유효하지만 결과를
절대 침하로 읽으면 안 된다.

GNSS 조회가 실패해도 번들 생성은 계속된다. GNSS 는 근거를 더해 주는 것이지 SLC 처리의
전제조건이 아니다(대신 그 사실을 매니페스트에 남긴다).

## 9. 전체 파이프라인 실행 — Windows

CV(영상/지오 영상) + InSAR(real, Track 정합) + PINN + FRAM 핫스왑.

```bash
python -m inframon --demo \
  --engine cv=real --engine insar=real --engine pinn=real --engine fram=real \
  --insar-source track.h5 --out data/project.h5
# Track 에 점별 고도가 없으면 DEM GeoTIFF 로 z 를 채움(geo 정합 필요):
#   ... --insar-source track.h5 --insar-dem dem.tif --out data/project.h5
```

- **좌표 정합**: CV 가 GeoTIFF/override 로 `geo_transform`/`crs` 를 주면 Track world 좌표를
  CV 픽셀에 정합(world xyz·DEM z·고정단 거리). 없으면 픽셀 identity 폴백.
- **고도 z 출처**: `track_height`(Track 점별 고도) > `dem_raster`(`--insar-dem` GeoTIFF 샘플,
  world↔DEM CRS 재투영·nodata/범위밖은 중앙값 메움) > `zero`. `insar_source` attr 의
  `z_source`/`dem` 에 출처·DEM 통계 기록. DEM 샘플은 geo 정합 경로에서만(픽셀 폴백은 z=0).
- Track 좌표가 경위도(EPSG:4326)면 CV 가 투영 CRS(예: EPSG:5179) geo_transform 을 줘야 하고
  pyproj 가 재투영한다(doctor 의 `crs_reprojection` 확인).
- **정확도 보정(opt-in, `--insar-corrections`)**: PINN/FRAM 이 소비하기 전에 LOS 시계열을 보정하고
  `/insar/velocity_mm_yr` 에 보정 속도를 저장한다. `insar=real` 과 `--import-track-h5` 양쪽 적용.
  적용/skip 내역은 `insar_source` attr 의 `corrections` 에 기록된다.
  ```bash
  # 검증된 경로 — 맨 SARvey/MintPy 산출물 인제스트(CV 지오참조 불필요)
  python -m inframon --import-track-h5 track.h5 --out data/project.h5 \
    --insar-corrections --insar-dem dem.tif \
    --insar-thermal --insar-temp-csv temps.csv   # 또는 --insar-fetch-temp (ERA5, 키 불필요)
  # 전체 체인에도 같은 플래그가 붙지만, 이 경로는 Track↔CV 좌표 정합이 필요하므로
  # 지오참조된 CV 입력(GeoTIFF/override)이 없으면 점이 ROI 밖으로 떨어져 0점이 된다.
  ```
  1. **공통성분(APS) 제거** — 단일 기준점이 아니라 결맞음 ≥ `--ref-min-coherence`(기본 0.9) 안정점의
     **에폭별 중앙값**을 뺀다(단일점 노이즈 주입 방지). **이득 가드**: 시간변동(std)이 실제로 줄 때만
     채택하고, 안 줄면 원본 유지 + `reference_skipped` 사유 기록. SARvey 는 내부적으로 이미 APS
     필터링(`filtering.apply_aps_filtering`)하므로 그 출력에선 대개 자동 skip 된다(무해). raw
     MintPy/STAMPS 출력엔 이득이 있다.
  2. **고도상관 성층대기 보정** — 점별 z 가 있어야 engage(`track_height` 또는 `--insar-dem`).
     z 스프레드 < 1 m 이면 skip. SARvey 의 `dem_error` 를 상부구조 고도로 쓰려면 58 변환기가 export.
  3. **열팽창 분리**(`--insar-thermal`) — `los = a + b·t + c·T` 회귀로 계절 열변형을 분리.
     온도원은 `--insar-temp-csv`(헤더 `date,temp_C`, 날짜는 `YYYYMMDD`/`YYYY-MM-DD`, **모든 취득일**
     이 있어야 함 — 결정론적이라 우선) → 없으면 `--insar-fetch-temp` 로 ERA5(Open-Meteo, 키 불필요,
     네트워크 필요). 공간 균일한 열신호는 1번에서 이미 빠지므로 이 단계는 **공간변이 잔차 열변형**
     (교량 α·L·ΔT)을 잡는다.
- **증분 재개**: 입력 안 바뀐 단계 생략 — `--resume`(+`--force-stage <stage>`).
- **상부거더 전체 변위장(가상센싱)**: `pinn=real` 이 InSAR 관측점(희소)에서 학습한 연속장을
  거더 종축의 촘촘한 가상센서(V개, `cfg.pinn_virtual_sensors` 기본 200)로 재평가해 **관측이
  없는 위치까지 포함한 거더 전체 변위량**을 낸다(계약 `vsens_*_ds`/`n_virtual`). API
  `GET /api/v1/bridges/{id}/pinn/girder-displacement` 로 종단 변위 프로파일·첨두·중앙경간 조회.
- **상판 전체 면 변위 지도(2D 가상센싱)**: 점 구름 PCA 로 상판 격자(G=`pinn_deck_long`×
  `pinn_deck_trans`, 기본 60×9)를 세워 각 격자점에 1D 전체변위장(고정단거리 평가)+관측 잔차
  IDW 2D 보간 → **상판 전역(관측 없는 위치 포함) 변위량**(계약 `deck_*_ds`/`n_deck`, 격자점
  world 좌표). API `GET /api/v1/bridges/{id}/pinn/deck-displacement`(격자 lat/lon+변위 히트맵).

## 정확도 게이트 (실데이터 평가 대상)

합성으로는 G2(부재 mIoU·축선)·G5(Morandi ROC-AUC=0.946·isotonic)가 통과됨. **실 수치 게이트는
실데이터·실 제원·실 가중치가 들어와야 평가**:

| 게이트 | 기준 | 필요 |
|---|---|---|
| G2 CV | bridge mAP@0.5≥0.7, 부재 mIoU≥0.6, 축선<3° | 실영상 + Transformer/SAM 가중치 |
| G3 InSAR | LOS RMSE(GNSS 대비), temporal_coherence>0.7 | 실 SLC + 기준점 |
| G4 PINN | forward L2<1%, EI 오차<10% | 실 제원(휨강성) |
| G5 FRAM | ROC-AUC≥0.9 | 실 손상/붕괴 사례 라벨 |
