# 데스크톱 앱 (더블클릭 실행)

대시보드(Streamlit)를 **주소창 없는 전용 창**에 띄워 일반 프로그램처럼 쓰는 방법.
지금은 로컬 단독 실행용이며, 같은 대시보드를 서버에서 띄우면 그대로 다중 사용자(브라우저 접속)로 확장된다.

## 1. 소스에서 바로 실행 (개발용)

```powershell
pip install pywebview          # 최초 1회
pip install -e ".[dashboard]"  # 대시보드 의존성
python -m inframon --app       # 전용 창으로 대시보드 실행
```

또는 저장소 루트의 **`inframon_app.bat` 더블클릭**.

## 2. 단일 실행파일(.exe) 빌드 (배포용)

파이썬이 없는 PC에서도 더블클릭으로 실행되는 폴더형 배포본을 만든다.

```powershell
pip install pyinstaller
pyinstaller inframon.spec --noconfirm --clean
```

- 산출물: **`dist/inframon/inframon.exe`** (폴더 `dist/inframon/` 전체가 한 세트)
- 배포: `dist/inframon/` 폴더를 통째로 압축해 전달 → 받는 사람은 폴더 안 **`inframon.exe` 더블클릭**
- 콘솔 없이 깔끔하게 하려면 `inframon.spec` 의 `console=True` → `False` 로 바꿔 재빌드

### 동작 구조
- `_app_entry.py` 가 진입점. 인자 없이 실행 → 데스크톱 창(`run_app`).
- 런처가 자기 자신을 `inframon.exe --_run-streamlit PORT` 로 재실행해 Streamlit 서버를 띄우고,
  pywebview 창으로 감싼 뒤, 창을 닫으면 서버도 정리한다.

### 빌드 메모 / 한계
- 데모 대시보드 용량 절감을 위해 `torch·transformers·rasterio·fastapi` 등은 **번들에서 제외**(`inframon.spec` 의 `excludes`).
  따라서 .exe 안에서는 **stub 파이프라인·데모 데이터**가 동작한다. 무거운 real 엔진(PINN real 등)은 소스 환경에서 실행.
- Streamlit/plotly/folium 은 런타임에 메타데이터·정적자산을 조회하므로 `collect_all`+`copy_metadata` 로 동봉한다(spec 참고).
