# inframon 전체 워크플로우 (KAIA) — 단계별 현황

KAIA 과제(InSAR·PINN → VLM 안전성 평가 → 지식그래프 → 보수보강)에서 inframon(우리 파트)의
end-to-end 흐름과 **지금 어디까지 됐는지**를 한 문서로 정리한다. 각 단계에 실행 명령과
현황 마커(✅완료 / 🔄진행중 / ⏸대기 / 🔴외부의존)를 붙였다.

> 기준 시점: 정자교 실 Sentinel-1 데이터 투입 중. 전체 285 테스트 green.

---

## 0. 큰 그림 (KAIA 전체)

```
[우리 — inframon] ━━━━━━━━━━━━━━━━━━━━━━━━━━━┓        [타 팀 — VLM]
 InSAR 변위 → PINN 가상센싱 → 통일 데이터 패키지 ━━━╋━→ + CAD + 정밀안전보고서
   (수직·종축)   (전체변위·EI·성분)  (CSV·JSON·figures)  ┃         ↓
                                                         ┃   대한민국 시방서·법규 평가
                                                         ┃         ↓
                                                         ┃   지식그래프 → 보수보강 추천
```
- **우리 책임**: 위성 변위 추출 + 구조해석 + VLM 이 삼킬 통일 데이터. **안전 최종판정은 VLM+시방서.**
- **1차년도 목표**: InSAR+PINN 을 Bmaps 에 통합 — **소프트웨어 완성, 실데이터 관통만 진행 중.**

## A. InSAR 선별 (A~E) — Windows · ✅ 실데이터 검증 완료

지도 위치 → 교량 확인 → 위성 데이터 선별 → 처리 번들. **다운로드·WSL2 불필요(검색만).**

| 단계 | 명령 / 모듈 | 현황 |
|---|---|---|
| A·B 교량 타깃(OSM) | `osm_bridge.confirm_bridge` → bbox·길이·**절점 폴리라인** | ✅ 정자교 실조회 |
| C·D SLC·트랙 선별(ASF) | `slc_search.search_slc/select_track` | ✅ 41장면·상승 path127 |
| E ERA5 master | `era5_master` (Open-Meteo, 키 불필요) | ✅ |
| **조건 게이팅** | `--insar-conditions data/jeongjagyo_real` | ✅ G1 g=0.41·G2 단일궤도 |
| F준비 번들 | `--make-sarvey-config data/jeongjagyo_real` | ✅ manifest+config |

산출: `data/jeongjagyo_real/{bridge_target,track_selection,selection_criteria,master_selection_era5,
processing_manifest,sarvey_config}.json`. **교량 형식별 SARvey 세분**(거더/트러스/아치/가동/케이블) 반영.

## C. SAR 처리 (F) — WSL2/Linux · 🔄 진행 중 (당신)

ISCE2·MiaplPy·SARvey 는 Windows 불가 → WSL2. `scripts/wsl_sarvey/` 스크립트(실 granule 박힘).

```bash
cd /mnt/d/프로그램
RECIPE=data/jeongjagyo_real
WORK=data/jeongjagyo_real          # ★ SLC 가 이미 여기(/SLC) → 재다운로드 방지

bash scripts/wsl_sarvey/00_setup_env.sh                 # 최초 1회: conda env(isce2/miaplpy/sarvey)
bash scripts/wsl_sarvey/10_download.sh   $RECIPE $WORK  # 궤도(.EOF)·DEM·aux (SLC 스킵)
bash scripts/wsl_sarvey/20_stack_isce.sh $RECIPE $WORK  # ◀ 지금 여기: 코레지스트레이션·간섭도
bash scripts/wsl_sarvey/30_miaplpy.sh    $RECIPE $WORK  # phase-linking → slcStack/geometryRadar.h5
bash scripts/wsl_sarvey/40_sarvey.sh     $RECIPE $WORK  # SARvey MTI → outputs/*_ts.h5
python3 scripts/wsl_sarvey/50_export_to_inframon.py \
  --sarvey-h5 $WORK/sarvey/outputs/*_ts.h5 --out $WORK/track.h5   # → Track H5
```

**진행을 시각적으로 보기** (별도 터미널):
```bash
python -m inframon --insar-progress data/jeongjagyo_real --recipe data/jeongjagyo_real --watch 5
```
→ 단계별 ✅/🔄/⬜ + 코레지 N/5 진행바 실시간 갱신.

> **현황: SLC 5/5 완료, 궤도·DEM 대기(10단계) → 20단계 진행 예정. 전체 진행판 20%.**
> ⚠️ 5장으론 20단계(스택·간섭도)까지 검증 가능. 30·40(시계열)은 장면 부족할 수 있음.

## D. 인제스트 + 4대 엔진 (G + 핫스왑) — Windows · ⏸ track.h5 대기

track.h5 가 나오면 **이 아래는 전부 자동**(제가 잇는 부분).

```bash
# 1) 투입 사전검증
python -m inframon --check-track data/jeongjagyo_real/work/track.h5     # ready=0/not=1

# 2) 전 엔진 real 핫스왑 → CRI
python -m inframon --demo \
  --insar-source data/jeongjagyo_real/work/track.h5 \
  --out data/jeongjagyo_real/project.h5 \
  --engine cv=real --engine insar=real --engine pinn=real --engine fram=real
```
- CV(ROI·부재·지오) → InSAR(정합·변위) → PINN(전체변위·EI·성분) → FRAM(CRI·경보)
- ✅ 코드·합성검증 완료, ⏸ 실 track.h5 입력 대기

## 지금까지 도출한 내용 (핵심 발견·결론)

실데이터·검증 과정에서 확인된 공학적·구현적 결론들:

1. **정자교는 단일 상승궤도로만 관측 가능** — 실 ASF 질의: 상승 path127 **41장면** 풍부, 하강은
   **2장면뿐**. → asc+desc 연직(처짐·침하) 분리 불가, 단일궤도 LOS·종축만. (조건 G2)
2. **상승궤도로 정자교 열팽창(종축) 관측은 가능** — 교량 축선 **29°**(OSM 절점 PCA), 상승 LOS(78°)
   대비 종축 민감도 **g=0.41**(임계 0.30 초과 → pass). bbox 종횡비 근사(g=0.13)는 **틀린 경보**였음. (조건 G1)
3. **변위 단위 1000배 버그 발견·수정** — `los_ds` 는 계약상 mm 인데 Bmaps API 가 ×1000(m→mm 오인)
   → value_mm 가 −26412mm 처럼 부풀려짐. 모든 생산자(합성·track import·융합)가 mm 임을 확인하고 제거.
4. **FRAM CRI 공간 국소화 한계** — 단일궤도 deprojection·강한 계절성분이 종축 CRI 의 침하 국소화를
   흐림(침하 vs CRI 상관 −0.08). asc+desc 연직 소비(opt-in)로 보완 시 **−0.08 → +0.52** 로 복원.
5. **5장으론 SARvey 시계열 불가** — phase-linking 은 ~15~20장 필요. 5장은 다운로드 인증 + ISCE2
   스택·간섭도(20단계)까지의 **배관 검증용**. 본 시계열은 디스크 확보 후 15장+ 재처리.
6. **선별·조건·번들 파이프라인은 실데이터로 완전 동작** — 다운로드·WSL2 없이 ASF·OSM·Open-Meteo
   실질의로 정자교 41장면 선별→14조건 게이팅→SARvey 번들까지 관통.
7. **교량 형식별 SARvey 차등이 실제로 다름** — 트러스(P1 0.88·dem_error 150·layover high)·아치(받침부
   기준)·가동(이음 세그먼트)이 거더 기준과 분기되어 config 로 흐름.

## 아직 미흡한 상황 (gap·한계·리스크)

### 데이터 부족 (코드는 되나 입력 없음)
- 🔴 **실 SLC 장면 부족** — 5장만 받음(시계열엔 15장+ 필요). 디스크 154→115GB 로 ~15장이 한계.
- 🔴 **실 교량 제원(휨강성 EI)** — PINN 절대 EI 게이트(G4: 오차<10%) 평가 불가.
- 🔴 **GNSS/현장 계측 기준** — InSAR LOS RMSE 검증(G3) 불가.
- 🔴 **CAD 도면·기존 정밀안전보고서** — VLM 입력(타 팀 합류분), 우리 패키지엔 미포함이 정상.
- 🔴 **국내 교량 시방서·법규 코퍼스** — KAIA 5단계(VLM 평가)용, 타 팀.

### 환경/외부 의존
- 🔴 **WSL2 처리 미완** — ISCE2/MiaplPy/SARvey 실행은 사용자 WSL2. 20단계 결과 대기 중.
- 🔴 **Bmaps 합의 3건** — bridge_id 체계·지도 SRS·인증 방식(KICT 결정 대기).
- ⚠️ **WSL 스크립트 `TODO(verify)`** — 도구 CLI/옵션은 버전 의존. 실행 시 본인 설치 버전과 대조 필요.

### 구현상 알려진 한계 (개선 여지)
- ⚠️ **FRAM CRI 국소화는 단일궤도·강계절에서 약함** — 연직 소비(opt-in)로 완화하나, 근본적으로
   asc+desc 가 있어야 침하 국소화가 견고. 정자교는 단일궤도라 이 혜택 못 받음.
- ⚠️ **PINN 절대 EI 는 합성서 강체 상한 포화** — 분포하중 휨이 약한 합성 데이터 한계. 실데이터 필요.
- ⚠️ **축선 PCA 는 OSM 절점 의존** — 절점 없는 옛 레시피는 bbox 종횡비 폴백(부정확). 곡선 교량은 PC1 근사.
- ⚠️ **입사각 G3 는 'unknown'** — 공칭 39° 가정. 실 입사각은 처리 후 Track H5 `incidenceAngle` 로 재평가.
- ⚠️ **KAIA 5~7(시방서·KG·보수보강) 미착수** — 우리 범위 밖(VLM 팀)이나, 우리 데이터가 그 입력.
