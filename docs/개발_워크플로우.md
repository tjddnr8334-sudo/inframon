# inframon 통합 개발 워크플로우 — STUB→실구현 전체 로드맵

본 문서는 4대 엔진(CV→InSAR→PINN→FRAM)과 통합 레이어의 5개 조사 결과를 하나의 실행 가능한 개발 로드맵으로 종합한다. 핵심 전제는 **contracts(Pydantic + project.h5)가 모듈 간 유일 통로**라는 점이며, 이를 안정점으로 삼아 STUB을 핫스왑 방식으로 실구현으로 교체한다.

---

## 1. 전체 개발 흐름 한눈에 (Phase 0 → Phase 5)

```
┌──────────────────────────────────────────────────────────────────────────────────┐
│  데이터 흐름(불변):   CV ──▶ InSAR ──▶ PINN ──▶ FRAM ──▶ 경보(CRI)                  │
│  통로(불변):          contracts/  project.h5  (/cv /insar /pinn /fram 그룹)         │
│  피드백(Phase5):      FRAM.critical_members ──▶ CV.grid_density / InSAR 점밀도 상향 │
└──────────────────────────────────────────────────────────────────────────────────┘

PHASE 0  [완료] 골격 + 전 STUB + pytest 3/3
   └ 입력: 없음        출력: 더미 CRI       게이트: ✔ 통과(현 상태)

PHASE 1  [통합 선행] 계약 동결·버전·검증 / 골든 회귀 / 어댑터 셀렉터 / conda·CI 매트릭스
   ├ 입력: 현 STUB 산출물(seed=42)
   ├ 출력: schema_version, validate(), cfg.engines{stub|real}, 녹색 CI, 골든 베이스라인
   └ 게이트 G1: 전 STUB CI 통과 + 형상/dtype/물리범위 회귀 불변식 통과 + 지연 import 강제

PHASE 2  [CV 실구현] YOLO11-OBB 탐지 → SAM2 분할 → 축선 PCA/Hough → sarsen layover/shadow → 격자밀도
   ├ 입력: SAR진폭/광학/DEM GeoTIFF + 궤도메타
   ├ 출력: 실 CVOutput(roi/member/grid_density/shadow/layover/geometry+crs+geo_transform)
   └ 게이트 G2: bridge mAP@0.5≥0.7, 부재 mIoU≥0.6, 축선오차<3°, shadow/layover non-None, 합성 fallback 회귀 유지

PHASE 3  [InSAR 실구현] (외부전처리) ISCE2/HyP3 → MintPy 어댑터 → 시계열역산 → MiaplPy/SARvey PS/DS → CV연동 격자 → LOS분해
   ├ 입력: Sentinel-1 SLC 스택(≥15장) + CVOutput(ROI/member/grid_density/shadow/layover/azimuth)
   ├ 출력: 실 InSAROutput(los/longitudinal/dates/coherence/member/l_from_fixed + geometry메타·velocity)
   └ 게이트 G3: LOS속도 RMSE(외부기준 대비), temporal_coherence>0.7 점비율, 종방향 복원오차 측정, 계약 형상 일치

PHASE 4  [PINN 실구현] 비차원화 → autograd 4차PDE → forward → 역산 EI → 4성분 분해 → 응답/V 산출 → STUB교체
   ├ 입력: InSAROutput(longitudinal[N,M], l_from_fixed[N], member[N], dates[M], coherence)
   ├ 출력: 실 PINNOutput(comp_*, strain/stress, EI/alpha, natural_freq, V_*, V_func_series, +point-wise)
   └ 게이트 G4: forward L2<1%, FEM기지 EI 역산오차<10%, V∈[0,1], 주입손상 V_anomaly 단조상승, CI<60s

PHASE 5  [FRAM 고도화 + 통합 실시간/피드백] 6측면 매핑 → 함수망 → 공명 재정식화 → 캘리브레이션 → Morandi 검증 → 모드2/3 + 피드백루프
   ├ 입력: PINNOutput(+point-wise V) + InSAR coherence + CV centerline
   ├ 출력: CRI확률(isotonic), 리드타임, 동적 함수망, FastAPI 엔드포인트, Prefect 실시간, 피드백 closed-loop
   └ 게이트 G5: ROC-AUC≥0.9, Morandi 붕괴前 경보+리드타임, 피드백 ≤max_refine 수렴, 모드2/3 e2e 통과
```

| Phase | 담당영역 | 핵심 의존(상류) | 병렬 가능? | 소요 | 난이도 |
|---|---|---|---|---|---|
| 1 | 통합 | (전 엔진 선행) | — (선행 단독) | 4~5주 일부 | 중 |
| 2 | CV | 없음(진입점) | InSAR/PINN/FRAM STUB와 병렬 가능 | 8~11주 | 상 |
| 3 | InSAR | CV(강결합) | PINN/FRAM STUB와 병렬 | 10~14주 | 최상 |
| 4 | PINN | InSAR(강결합) | FRAM STUB와 병렬 | 8~12주 | 상 |
| 5 | FRAM+통합 | PINN(강), InSAR/CV(약) | 통합 인프라는 Phase1부터 선행 | 7~9주 + 통합 10~14주 | 중상 |

## 2. 권장 개발 순서 & 병렬화 전략

데이터 흐름은 `CV→InSAR→PINN→FRAM` 단방향이지만, **계약(*_ds HDF5 규약 + run_xxx(store, 입력계약, cfg) 시그니처)을 통로로 하므로 다운스트림은 업스트림 STUB을 mock으로 쓰며 병렬 개발이 가능**하다.

**임계경로(최장): 통합 Phase1 → InSAR(최난, 10~14주) → PINN → FRAM 검증.** InSAR가 전체 일정을 좌우하므로 가장 먼저 인력·외부환경(WSL2/HyP3)을 확보한다.

```
주차  0────4────8────12───16───20───24───28───32───
통합  ████████ G1                          ██(모드2/3)██ G5   ← 계약/CI는 반드시 선행
CV       ████████████ G2                                     ← 진입점, 즉시 착수
InSAR        (CV계약 합의후)████████████████ G3               ← 외부 전처리 환경 병행 구축
PINN              (InSAR STUB mock)██████████ G4              ← InSAR 계약 합의되면 STUB로 병렬
FRAM                   (PINN STUB mock)████████ G5            ← V_func_pointwise 계약 합의 선행
```

**병렬개발 3원칙**
1. **통합 Phase1을 모든 엔진보다 먼저** 안정화한다(계약 동결·골든 회귀·셀렉터). 이것이 4모듈 독립 개발의 안정점.
2. **계약 확장은 사전 합의 후 Optional 필드로만** 추가(하위호환). InSAR↔PINN의 단위/부호/epoch, PINN↔FRAM의 `V_func_pointwise_ds` 신설은 머지 전 합의 필수.
3. **다운스트림은 상류 STUB으로 mock 개발**하다가 실구현 도착 시 `cfg.engines={'insar':'real'}`로 핫스왑.

## 3. Phase별 상세

### Phase 1 — 통합 인프라 선행 (계약·CI·회귀 안정화)

**목표:** STUB↔실구현 핫스왑이 가능한 안전판(계약 버전·형상검증·골든 회귀·셀렉터·conda CI)을 먼저 구축해 이후 4모듈을 독립·병렬로 교체 가능하게 한다.

**작업 단계(산출물):**
1. 계약 동결: 각 Output에 `schema_version:str="1.0"` 추가, `ProjectStore.write_meta`가 `.attrs` 기록, `read_meta`가 major 불일치 시 raise. (→ 단위테스트)
2. `ProjectStore.validate(group)` 추가 — `los[N,M]`, `CRI[N,M]`, `V_func_series[n_func,M]` 등 shape/dtype 검사. (→ 위반 시 raise 테스트)
3. 골든 회귀 베이스라인: 현 STUB(seed=42)의 CRI/V/comp_*를 pytest-regressions로 스냅샷. (→ `tests/regression/`)
4. 엔진 셀렉터: `cfg.engines={'cv':'stub'|'real',...}`, orchestrator가 레지스트리에서 구현 선택. 무거운 의존성(torch/gdal/ultralytics)은 **함수 내부 지연 import**. (→ stub/real 혼합 실행 CI)
5. conda+pip extras 매트릭스: `environment.yml`(gdal/rasterio/snaphu=conda-forge) + extras(cv/pinn/insar/dashboard), GitHub Actions `windows-latest`, ruff+mypy+pytest 게이트, `conda-lock.yml`. (→ 녹색 CI 배지)
6. 데이터관리 기초: ProjectStore 압축 gzip→Blosc-zstd(hdf5plugin), DVC remote 설정, `.gitignore`+`.dvc`.

**라이브러리:** pydantic v2, h5py + hdf5plugin/Blosc, DVC, pytest + pytest-regressions, ruff + mypy + pre-commit, GitHub Actions(windows + conda), conda-lock.

**contracts 변경:** 각 Output에 `schema_version="1.0"`; `FeedbackRequest(critical_members, refine_density, iteration)` 신규; `PipelineConfig`에 `engines:dict[str,str]`, `max_refine:int`, `cri_converge_eps:float`; `io.py`에 `validate()` + 압축 cfg 선택. (`GROUPS`/`*_ds` 규약·`MEMBER_TYPES` 불변)

**함정:** ① 최상단 import로 코어 STUB·CI 붕괴 → 지연 import 강제. ② Windows에서 gdal/rasterio/snaphu pip 설치 실패 → conda-forge 필수. ③ HDF5 동시 쓰기 취약 → 단일 writer 직렬화(파일락/Prefect concurrency). ④ pytest-regressions가 BLAS/torch 비결정성으로 깨짐 → 형상·물리범위·단조성 불변식 위주 + seed 고정 + rtol. ⑤ 대용량 .h5 git 직커밋 금지(DVC).

**검증:** 계약 형상/차원 100% 일관, 물리범위(V∈[0,1], CRI∈[0,1], isfinite) 위반 0, 전 STUB CI 통과, extras 조합 import 스모크 통과.

**소요·난이도:** 4~5주(계약강화 1.5 / 셀렉터 1 / conda·CI 1.5 / HDF5·DVC 1.5), 난이도 중.

> **검증 게이트 G1:** 전 STUB CI 녹색 + `validate()` 형상/dtype 위반 raise + 셀렉터로 stub/real 혼합 실행 성공 + 지연 import로 코어만 설치해도 전 STUB 통과. 미통과 시 엔진 실구현 착수 금지.

### Phase 2 — CV 실구현 (ROI 자동산정)

**목표:** SAR진폭+광학 융합 영상에서 교량 자동탐지(YOLO11-OBB)·부재 분할(SAM2)·축선(PCA/Hough)·DEM기반 layover/shadow·부재별 적응형 격자밀도를 산출해 지오레퍼런스된 실 CVOutput을 생성. InSAR의 PS/DS 격자·부재 라벨 prior가 된다.

**작업 단계(산출물):**
- S0 계약 고정 + 테스트 골격: `shadow_ds/layover_ds`(현 None) 실제 채움, `run_cv`를 detect→segment→axis→radar_mask→grid 5함수로 분해, 합성 fallback 유지. (→ 기존 3테스트 통과)
- S1 융합 입력: rasterio로 SAR/광학 EPSG:5179 정합, 4채널 early-fusion, 위상상관 미세보정, 구름 시 SAR 단독. (→ `load_fused_input()`, 정합 RMSE<2px)
- S2 탐지: YOLO11x-obb(DOTA 'bridge') + SAHI 슬라이스추론, OBB 회전각=방위각 초기추정. (→ `detect_bridge()`)
- S3 부재 분할: OBB→SAM2 박스프롬프트로 roi_mask, 부재 의미부여는 기하규칙(끝단=abutment, 직교 주기구조=pier, deck∩pier=bearing)+소량 라벨 YOLO11-seg 약지도. (→ `segment_members()` MEMBER_TYPES별 마스크)
- S4 축선: `cv2.PCACompute` 1주성분=종축, HoughLinesP 교차검증, affine으로 지리좌표 centerline·length·width. (→ `compute_geometry()`, PCA·Hough 오차<3°)
- S5 layover/shadow: sarsen으로 DEM+궤도 입사각 기반 2-pass 마스크, 메타 부재 시 기하근사+신뢰도 플래그. (→ `radar_masks()`)
- S6 적응형 격자밀도: 기본1.0, pier/bearing/abutment 2.0~3.0, shadow=0, layover=0.5, 0~1 정규화 옵션. (→ `build_grid_density()`)
- S7 학습데이터: DOTA/iSAID/xView 전이학습 → 국내 소량 라벨(Roboflow/CVAT) → albumentations 증강(SAR 스펙클/광학). (→ `.pt` 가중치, mAP 베이스라인)
- S8 통합·CI: GPU/CPU 분기, lazy-load 캐시, 골든 1건 + 합성 fallback 회귀, Windows torch/CUDA 휠 확인.

**라이브러리:** ultralytics(YOLO11/SAM2), segment-geospatial(samgeo), rasterio+GDAL, sarsen, OpenCV, scikit-image+SAHI, shapely+geopandas, albumentations.

**contracts 변경:** `CVGeometry`에 `crs:str="EPSG:5179"`, `geo_transform:tuple`(affine 6원소) 추가; `CVOutput`에 `incidence_angle:float|None`, `member_confidence:dict|None`; `PipelineConfig`에 `sar_path/optical_path/dem_path/orbit_meta/crs/device/yolo_weights/sam_weights`. (`MEMBER_TYPES` 불변, io.py 불변)

**함정:** ① SAM2는 의미라벨 모름 → 기하규칙+소량 미세조정 약지도. ② DOTA 'bridge' 도메인갭 → 전이학습 필수, zero-shot 과신 금지. ③ SAR/광학 정합오차(고도차 시차) → DEM 보정+phase-correlation, 불가영역 단독분기. ④ layover/shadow는 정확 궤도메타 필요 → 없으면 명시적 다운그레이드. ⑤ Windows+py3.11 sam2 빌드 → ultralytics 내장 SAM2 + CPU fallback. ⑥ 대형타일 OOM → SAHI + 경계 OBB NMS. ⑦ 곡교 PCA 단일축 오류 → piecewise PCA/skeleton+spline.

**검증:** bridge mAP@0.5≥0.7(전이후), 부재 mIoU≥0.6, 축선 azimuth 오차<3°, layover/shadow는 입사각 변화에 이동방향 일관(육안 교차), 합성 골든 해시 회귀 유지.

**소요·난이도:** 8~11주, 난이도 상. S3 부재분할(2.5주)·S7 라벨확보(2주, 임계경로)가 변수. 라벨 외주 지연 시 +2~3주.

> **검증 게이트 G2:** mAP@0.5≥0.7 & mIoU≥0.6 & 축선오차<3° & CVOutput의 shadow/layover/grid_density 모두 non-None & 합성 fallback 회귀 통과 & orchestrator가 채워진 CVOutput으로 InSAR까지 흐름. **InSAR S5(좌표정합) 착수 전 CV 담당과 EPSG:5179·affine·MEMBER_TYPES 인덱싱 규약 합의 필수.**

### Phase 3 — InSAR 실구현 (변위 추출, SARvey 상위호환) — 최난

**목표:** 검증된 오픈소스(ISCE2 topsStack → MiaplPy 위상연결 → MintPy/SARvey 시계열역산)를 어댑터로 래핑해 PS/DS 변위 시계열을 추출, CV ROI/축선/격자밀도를 적응형 격자로 주입, LOS를 종방향 성분으로 분해해 `/insar`에 기록. STUB 합성변위를 실 위성관측으로 1:1 교체.

**작업 단계(산출물):**
- S1 어댑터+데이터 확보: `insar/adapters/` 신설, `InSARProductSet` 추상화, asf_search로 동일트랙 S1 IW SLC 15~30장. **Windows 제약상 ISCE2 로컬빌드 대신 ASF HyP3(클라우드)를 1차 기본**, ISCE2/SNAP 로컬은 옵션. (→ `download_s1.py`)
- S2 다중포맷→MintPy: `prep_isce/prep_snap/prep_gamma/prep_hyp3`를 호출하는 `build_mintpy_inputs(processor=…)`로 분기, `inputs/ifgramStack.h5`로 정규화. (→ `mintpy_prep.py`)
- S3 시계열역산 래핑(MintPy) — **STUB 1차 교체**: `smallbaselineApp.py`→`timeseries.h5`/`velocity.h5`/`temporalCoherence.h5`. `run_insar()`를 '합성'→'MintPy 결과 판독→project.h5 매핑'으로 교체. (→ `mintpy_reader.py`)
- S4 PS/DS 위상연결: 좁은 거더용 MiaplPy `miaplpyApp.py`(SHP→phase linking→PS/DS), SARvey(`slcStack.h5`+`geometryRadar.h5`→`sarvey -f config.json 0 4`) 인프라모드. `backend='mintpy'|'miaplpy'|'sarvey'` 셀렉터. **PS/DS는 직접 구현 않고 래핑**. (→ `insar/backends/*.py`)
- S5 CV연동 적응형 격자: ROI 마스크→sarvey_mask/maskConnComp, grid_density로 부재별 점밀도 가중(거더=고밀도 PS, 교대=저밀도 DS), shadow/layover로 신뢰불가 제외, 각 점에 member·`l_from_fixed` 부여. **CV 좌표↔InSAR 지리좌표 정합이 핵심**. (→ `adaptive_grid.py`)
- S6 언래핑·품질·보정: snaphu 후 MintPy `unwrap_error_phase_closure`, temporal_coherence>0.7 마스킹, 대류권 보정(pyaps/ERA5) 옵션, 기준점 자동선정, SARvey 2단계 언래핑. (→ 보정 on/off 리포트)
- S7 LOS→종방향 분해: `e_los`를 incidence θ·heading으로 구성(geometryGeo.h5), CV azimuth로 축선 투영→`/insar/longitudinal`. asc+desc 둘다면 최소자승 3D(수직+EW), 단일트랙이면 근사임을 메타 명시. (→ `los_decompose.py`)

### Phase 4 — PINN 실구현 (구조건전성 PDE/ODE 엔진)

**목표:** InSAR 종방향 변위 시계열을 입력으로 Euler-Bernoulli 4차 PDE를 autograd 물리손실로 내재화한 신경망을 학습 — (1) 변위를 열/하중/침하/이상 4성분으로 물리분해 (2) EI·고유진동수·응력 역산 (3) 물리이탈도 V를 FRAM에 전달. STUB(최소제곱+np.gradient)를 물리추론으로 교체.

**작업 단계(산출물):**
1. 물리·계약 정합 + **비차원화**: `EI·d⁴w/dx⁴=q(x)`, `x̂=x/L, ŵ=w/w_ref`로 [0,1] 스케일(비차원화 없으면 grad 발산/NaN). PINNOutput 필드 1:1 매핑표. (→ `pinn/physics.py` docstring + 왕복변환 무손실 테스트)
2. autograd 빌딩블록: `beam_residual(net,x)`로 `autograd.grad(create_graph=True)` 2회→4차도함수, **order-reduction**(w→θ→M→Q 1차연쇄) 옵션, tanh/SiLU(ReLU 금지). (→ 해석해 vs autograd 4차 <1e-4)
3. 정적 forward PINN(EI 기지): MLP `w_θ(x̂)`, 손실=L_PDE+L_BC, **hard-constraint BC** 우선, Adam→L-BFGS 2단계. (→ 처짐 L2<1%)
4. 정적 역문제(EI 역산): EI를 부재별 `nn.Parameter`(member 그룹화), 손실=L_data+L_PDE+L_BC, DeepXDE inverse 패턴 자체구현. (→ FEM 기지 EI 회수 오차<10%)
5. 4성분 분해 네트워크: `w=w_thermal+w_load+w_settle+w_anomaly` multi-head, 물리제약(thermal=α·L·ΔT 계절성·l_from_fixed 비례, settle=시간 단조, load=PDE만족, anomaly=잔차). (→ 합 재구성 RMSE<노이즈, 주입손상 시 anomaly 상승)
6. 구조응답·고유진동수: `strain=d²w/dx²·c`, `stress=E·strain`, natural_freq는 EI·질량밀도 모드식/소규모 고유치. (→ 형상·단위 검증)
7. **변동 V 산출(★FRAM 인터페이스):** V_thermal=1−R², V_settle=정규화 침하속도, V_anomaly=anomaly 에너지비, V_load=PDE잔차. `V_func_series[4,M]` 물리성분 기반 교체, [0,1] 정규화 필수. (→ FRAM 무수정 동작, 기존 pytest 3종 통과)
8. STUB 교체 + 폴백: `run_pinn(store,insar,cfg)` 시그니처·반환 유지, cfg에 epochs/lr/layers/λ/use_pinn, device(cpu) 고정, NaN/미수렴 시 회귀분해 graceful fallback. (→ CI 저epoch <60s)
9. FEM 연계('데이터 생성기'부터): sfepy/FEniCSx로 기지 EI→변위 진실값 생성→역산 채점. L_FEM 결합은 Phase 2 후순위. (→ `pinn/fem_gen.py`)
10. 동적/열-구조/침하 확장(단계적, 계약 불변).

**라이브러리:** PyTorch 2.8(CPU, autograd 핵심), DeepXDE(검증·노트북 한정, 본체 미의존), sfepy/FEniCSx(FEM 진실값), scipy/torch L-BFGS, Optuna 또는 NTK/grad-norm self-adaptive weighting. **PhysicsNeMo는 미채택**(CUDA/대규모 지향, 본 소규모에 과함).

**contracts 변경:** PINNOutput 자체 불변(현 필드가 실출력과 호환). 선택 추가: `EI_unit/stress_unit`, `w_ref/L_ref`, `train_residual/converged:bool/n_epochs` 진단(FRAM 신뢰도 가중); `PipelineConfig`에 `pinn_epochs/lr/layers/loss_weights(λ_data,λ_pde,λ_bc,λ_ic,λ_fem)/use_pinn/pinn_device`; (Phase2) `deflection_future_ds[N,P]`.

**함정:** ① **4차도함수 NaN** → 비차원화 + order-reduction, ReLU 금지. ② 손실항 불균형(gradient pathology) → grad-norm/NTK self-adaptive weighting, 고정 λ 금지. ③ 역문제 비식별성(점별 EI 200개) → member 그룹공유 + 공간평활 정칙화. ④ 노이즈 과적합(σ=0.3, M=36) → L_data에 coherence 가중 + anomaly sparsity. ⑤ CI 성능 → use_pinn 저epoch/fallback. ⑥ **좌표축 혼동** → 빔축을 `l_from_fixed`(m)로 일관, InSAR와 단위 합의. ⑦ 프레임워크 과결합 → 코어는 순수 torch.

**검증:** (1) 단위물리: 해석해 forward L2<1%, 4차도함수<1e-4. (2) **역산 정확도(핵심):** FEM 기지 EI 회수 오차<10%, 손상위치 IoU. (3) 통합·계약: 기존 pytest 3종 무수정 통과, `V_func_series.shape==(n_func,M)`, V∈[0,1], 주입손상 V_anomaly·CRI 단조상승.

**소요·난이도:** 8~12주, 난이도 상. step1-3(비차원화+forward 수렴, 최대위험) 2~3주, step4-5(역산+분해, 비식별성·손실균형) 3~4주, step6-8 1.5~2주, step9 FEM 1~1.5주, step10 2주+(Phase2). 수렴이 안 풀리면 order-reduction→hard BC→NTK가중 순차 투입.

> **검증 게이트 G4:** forward L2<1% & FEM 기지 EI 역산오차<10% & V∈[0,1] & 주입손상 교각 V_anomaly 단조상승 & 기존 pytest 3종 통과 & CI 저epoch <60s. **FRAM과 `V_func_pointwise_ds[n_func,N,M]` 신규 계약·func_names 순서 합의 필수.**

### Phase 5 — FRAM 고도화 + 통합 실시간/피드백

**목표:** PINN 출력을 Hollnagel FRAM 6측면(I/O/P/R/C/T) 함수망으로 정식 매핑, V→공명 4종→CRI를 N-K 결합·Monte-Carlo로 정량화, 실 붕괴교량(Morandi)으로 임계·리드타임 캘리브레이션. 동시에 통합 레이어가 모드2(API)/모드3(실시간)·피드백 루프를 완성. (FRAM은 STUB가 아닌 NumPy 실구현 → '정식화·정량화·검증 고도화')

**작업 단계(산출물):**

*FRAM 고도화:*
1. 6측면 매핑: PINN→측면 귀속(Output=V_i, Input=상류 O, Resource=EI/alpha, Control=설계허용 대비 stress/deflection, Precondition=coherence/temporal_coherence, Time=취득시점). `[n_func,6,N,M]` 텐서. (→ `fram/aspects.py`)
2. 함수망 그래프(networkx): 물리인과(침하→받침→하중재분배→열응력) 엣지, betweenness/eigenvector centrality로 전파허브 식별, 결합행렬 C_ij. **무차별 corrcoef를 인과제약 결합으로 대체**. (→ `fram/network.py` nx.DiGraph)
3. 변동 V 정밀화: PINN 점별 성분에서 V_i(x,t) 직접 산출(현 전점평균 공간평탄 제거), max정규화→물리기준 정규화(설계허용 대비/z-score). (→ `V_func_pointwise_ds` 활용)
4. 공명 4종 재정식화: R_ij=scipy 윈도우 교차상관+인과지연+Spearman, A(증폭)=N-K 결합강도식(선형합 초과분), R_spatial=CV centerline 측지거리 기반 ∇·V, R_div=ruptures 변화점+가속도. (→ 4함수 회귀테스트)
5. 캘리브레이션: SALib Sobol로 w1~w4 민감도, 로지스틱회귀로 가중치 학습, isotonic regression으로 CRI→붕괴확률 보정, 고정 물리/통계 정규화. (→ `fram/calibration.py`, weights/thresholds JSON, ROC-AUC/PR)
6. 동적 함수망+리드타임: 시점별 망 스냅샷, ruptures 변화점으로 전조개시·리드타임. (→ `[T]` 직렬화)
7. **Morandi 역추적 검증:** Milillo 2019(COSMO-SkyMed/S1, 2017-03 변위 10→70mm/yr 급가속) 입력→붕괴(2018-08) 前 경보·리드타임, R_div가 급가속 포착. 원자료 비공개 시 논문수치 합성 재현(과대주장 금지). (→ `tests/validation/morandi_case.py`)
8. 육각 시각화: plotly Scatterpolar 6측면 레이더 + networkx→plotly 함수망, streamlit 탭 통합.

*통합 실시간/피드백:*
9. 모드2 API: 각 `run_xxx`를 `/cv /insar /pinn /fram` FastAPI 엔드포인트로 래핑(InSAR를 SARvey 대체로 단독호출), 백그라운드는 Prefect flow + run_id. (→ httpx e2e)
10. 모드3 실시간+피드백 루프: Prefect 파일도착/스케줄 트리거→증분 InSAR→PINN→FRAM, `warning.critical_members` 시 해당 부재 `grid_density` 상향·CV ROI 정밀화·InSAR 점밀도 증가를 다음 run cfg에 주입. **max_refine·CRI delta eps로 무한루프 방지, point_id 안정매핑.** (→ 피드백 1~2회 후 밀도상향·CRI갱신 통합테스트)
11. 경보·관측성: CRI 임계초과 시 이메일/웹훅, Prefect UI 관측, 마일스톤 게이트 워크플로. (운영 데스크톱 확정 시 PyQt6, 데이터레이어 불변)

**라이브러리:** networkx/igraph, numpy+scipy.signal/stats, SALib, scikit-learn(로지스틱·isotonic·ROC/PR), ruptures, plotly, pydantic, pandas/h5py / (통합) Prefect 3.x, FastAPI+uvicorn, ONNX Runtime(선택).

**contracts 변경:** PINNOutput에 `V_func_pointwise_ds[n_func,N,M]`(공간분해 R_ij 전제); FRAMOutput에 `aspects_ds[n_func,6,N,M]`/`coupling_matrix_ds`/`centrality_series_ds`/`precursor_onset_date:float|None`; FRAMWarning에 `cri_probability:float`/`precursor_change_points:list`; 신규 `CalibrationMeta`(weights/thresholds/ROC_AUC). 전부 `*_ds`경로·Field default로 하위호환. **PINN/orchestrator와 동시 머지.**

**함정:** ① FRAM 6측면 무비판 수치화 → 물리근거(설계허용·강성여유) 결부, 결합행렬은 인과제약 그래프(corrcoef 금지). ② V 전점평균이면 R_ij 공간정보 소실 → PINN 점별 계약 선행. ③ max정규화는 사례간 비교 불가 → 고정 기준. ④ 캘리브레이션 라벨 희소(Morandi 등) → 합성보강+교차검증, 과적합 주의. ⑤ Monte-Carlo 계산폭발 → 부재단위 클러스터링. ⑥ 리드타임을 '최초초과'로 정의하면 오경보 → ruptures+추세지속, false alarm rate 지표화. ⑦ (통합) HDF5 동시쓰기 → 단일 writer 직렬화. ⑧ 피드백 무한루프 → max_refine·eps + point_id 안정. ⑨ Prefect 조기도입 → 모드1은 순수함수, 실데이터 자동수신 시점에 도입.

**검증:** 주입손상 시 해당부재 CRI·R_div 단조증가/정상부재 불변; ROC-AUC≥0.9, isotonic 후 Brier 개선; SALib Sobol가 물리직관(R_div·A 우세)과 정합; Morandi 2018-08 前 경보+리드타임(개월), 2017-03 급가속을 R_div 변화점 포착; centrality 시계열이 foundation→bearing→load 전파 추적; 정상 시계열 false alarm rate로 운영점 결정. (통합) 피드백 ≤max_refine 수렴, 모드2 각 엔진 e2e, 모드3 파일도착→CRI갱신 지연 측정, 전 항목 미충족 시 머지 차단.

**소요·난이도:** FRAM 7~9주(6측면+망 2 / 공명재정식 2 / 캘리브 1.5 / 동적망 1 / Morandi 1.5 / 시각화 1), 통합 실시간/피드백 별도 약 5~7주(모드2 1.5 / 모드3+피드백 2.5 / 경보·게이트 1.5). 난이도 중상.

| # | 위험 | 영향 | 완화책 |
|---|---|---|---|
| 1 | **ISCE2/MiaplPy/SARvey가 Windows 네이티브 불가**(사실상 Linux 전용) — '윈도우 네이티브' 전제와 충돌 | InSAR(임계경로) 전체 지연·환경 붕괴 | 외부 전처리는 **WSL2/conda-forge(Linux) 또는 ASF HyP3 클라우드**로 분리, inframon은 **결과 H5만 소비하는 경계 설계**. HyP3를 1차 기본 경로로, ISCE2 로컬은 옵션. CI는 'cached/precomputed' H5 읽기전용 회귀로 외부툴 미실행. |
| 2 | **CV↔InSAR 좌표계 정합 실패**(영상픽셀/EPSG:5179 ↔ 레이더/지리 EPSG:4326) | member 라벨 전체 오정렬 → 적응형 격자·종방향 분해·EI 역산 무의미 | 첫 2주에 EPSG:5179·affine·MEMBER_TYPES 인덱싱 규약 **문서 합의**. CVGeometry에 `crs`/`geo_transform` 추가. S5 ROI 주입 전 geometry 기반 좌표변환 의무화 + 정합 RMSE 로깅. |
| 3 | **PINN 4차 PDE 수렴 불안정**(비차원화 없으면 grad 발산/NaN, 손실항 불균형) | Phase 4 최대 불확실성, 안 풀리면 V 산출 불가 | **비차원화(x̂∈[0,1]) + order-reduction(w→θ→M→Q) + tanh/SiLU(ReLU 금지)**. 손실은 grad-norm/NTK self-adaptive weighting(고정 λ 금지). Adam→L-BFGS. 미수렴 시 hard BC→NTK가중 순차 투입, 최종 회귀분해 graceful fallback. |

| 5 | **STUB↔실구현 핫스왑 시 계약/회귀 붕괴**(최상단 무거운 import, HDF5 동시쓰기, 부동소수 비결정성, 피드백 무한루프) | 4모듈 병렬개발 안정점 상실, CI 깨짐 | Phase1 선행으로 **schema_version·validate()·골든 회귀** 박제. 지연 import + 코어만으로 전 STUB CI 강제.
